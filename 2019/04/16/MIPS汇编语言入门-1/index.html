MIPS寄存器

下面列出比较重要的几个

  • zero
zero: 恒为0
eg: move $t0, $s0 ($t0=$s0) = add $t0, $zero, $s0 ($t0=0+$s0)
都执行了 $t0=$s0
  • a系列
a0-a3: 过程调用参数(在父函数给子函数传值的时候会用到)
eg: move $a0, $t0
        move $a1, $s0
        jal compare  
# compare(a,b)子函数里面用到了父函数里面的值，要先把他们传到a0，a1这两个寄存器里面
  • t系列 s系列
t0-t7: 临时变量，类似for (int i=1; i<=n; i++)这里面的i
s0-s7: 其实可以看做是全局变量（但实际上并不是这个意思，它指在调用过程中被保存）
  • gp sp fp
gp: 固定值 static data域的最高位 0x10008000

sp: 栈指针，说白了就是栈顶的位置，
    其实可以看做是一个参照点，是当前过程所存的数据的最低位，
    我们在取当前过程中用到的局部变量的值的时候是依据sp的位置来取的。
    在每次进入一个新的过程(void/int)的时候，sp的值都会变化（当函数过于简单的时候不用变）

fp: 帧指针，说白了就是栈底的位置，有的时候这个东西会没什么用
  • ra
ra: 过程调用的返回值，一般在程序开头存到内存里，因为有的时候要进入子过程，ra就又要变化

eg:
    set-array: addi $sp, $sp, -52   # 开辟空间
    sw   $ra, 48($sp)    # M[$sp+48]=ra，先存起来
    ...                  
    ...                  # 在这些过程中ra被改变
    ...
    lw   $ra, 48($sp)    # ra=M[$sp+48]
    jr   $ra             # 然后再返回到父函数，所谓过程调用的返回值

ra和jal这个汇编指令关联密切
jal compare = $ra=PC+4, go to compare
$ra=PC+4，目的在于子过程返回的时候，正好可以返回到jal命令的下一行命令去执行

MIPS 指令格式

  • R型
一系列的 运算操作以及

移位sll,srl:
sll $s1, $s2, 10 指 $s1=$s2<<10；

条件分支slt,slti:
slt $s1, $s2, $s3 指 if ($s2<$s3) $s1=1; else $s1=0；
  • I型
存储：lw,sw
运算：andi,ori,addi等
跳转：beq,bne
  • J型
直接跳转：j L
过程调用：jal compare
过程返回：jr $ra
