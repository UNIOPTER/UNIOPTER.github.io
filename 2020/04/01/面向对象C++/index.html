面向对象C++（一）

基本思想

  • 过程化程序设计和面向对象程序设计：

    过程化：数据和函数分开，缺点：出现大量全局变量、程序复杂、难以进行修改和扩充；

    OOP：精力集中设计对象上；

类基础部分

一个常见的类定义如下：

私有成员和公有成员

  • 私有成员：类外的程序不能访问私有成员，在默认情况下，类中的成员都是私有的；

  • 公有成员：类外程序可以访问

指向对象的指针和 “->”

内联函数成员

即完整定义在类内部的成员，下面SetData、ClacArea、GetArea均为内联函数成员。

构造函数和析构函数

  • 构造函数：在调用类之前先调用它，构造函数可以重载，但是一个类只能有一个缺省构造函数

  • 析构函数：当对象生存期结束时，自动调用析构函数进行善后处理，析构函数常用的操作是动态释放空间

类高级部分

静态成员

  • 是类中的一个成员，用static修饰；

  • 同一个类中所有对象共享该变量；一个类的静态成员不依赖于是否定义了该对象，即使程序中没有定义该类，这个静态变量依旧存在；

  • 静态变量必须在外面对其进行再次说明；

应用场景：一个预算类里面有一个静态成员用于计算总预算；

友元函数

  • 友元函数不是类中的函数成员却可以访问类中定义的私有成员

  • 友元函数可以是一个外部函数，也可以是另外一个类的函数成员

  • 函数func要访问类Class1里的成员，那么要在Class1这个类里面对func进行声明，说明func是Class1的友元

在func和Class2里面不需要做任何说明，直接访问Class1的私有成员即可

就是操与被操的关系，被操者要说明下自己的炮友是谁，而操的人没必要宣传

对象赋值问题

  • 赋值运算符“=”，执行按位赋值，浅复制（浅拷贝），当用一个对象去初始化另一个对象时，对象成员之间的赋值也是浅拷贝，当遇到指针的时候，浅拷贝会出问题；
  • 拷贝构造函数：是一种特殊的构造函数，当采用一个对象初始化另外一个对象的时候将自动调用该函数；

  • 调用拷贝构造函数的三个情况

    • 用对象初始化同类的另外一个对象，比如PersonInfo st1=st2;

    • 函数的形参是对象，当进行参数传递时，将调用拷贝构造函数，比如 void ChangePerson(PersonInfo p);，在主程序中调用该函数 ChangePerson(st1)；

运算符重载

  • 重载赋值运算符

    当对象中有指针成员，拷贝构造函数能解决对象初始化的问题但是解决不了处理对象赋值的问题

    this指针：隐含的内嵌指针，在函数成员中频繁出现，指向调用成员函数的对象

    运算符的重载，虽然可以改变运算符的含义，但是最好不要改变其原意

  • 重载双目运算符，如+，-，<，>等

  • 重载单目运算符，如++，–等

    前置++：先++，后操作

    后置++：先操作，后++

  • 重载流运算符

    必须通过友元函数的形式实现函数重载

    原因：cout<<feet1;，这里其实可以看做cout访问了feet1里面的内部成员，故要通过友元函数实现重载；

  • 重载类型转换运算符

对象组合
